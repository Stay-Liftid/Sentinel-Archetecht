# Contributing
## Sovereign Contribution Philosophy
**The Remote Viewer** operates on absolute privacy, decentralization, and zero-trust principles. Contributions from developers, security researchers, and archivists are welcomed strictly on a sovereign, peer-to-peer basis.
Pseudonymous and anonymous contributions are fully supported and encouraged to protect individual privacy against global observers and centralized state or corporate tracking.
### Contribution Guidelines
 1. **Identity Protection:** You are never required to link your real-world identity, corporate email, or personal repository handles to a contribution. Patches may be submitted via anonymous git configurations or encrypted bundles.
 2. **Cryptographic Verification:** All code contributions must be cryptographically signed using your independent PGP or Ed25519 keypair to maintain binary integrity and prevent unauthorized tampering.
 3. **Zero Telemetry & Tracking:** Submitted code must not contain any hardcoded telemetry, analytics hooks, remote procedure calls (RPC) to centralized endpoints, or hardware attestation triggers.
 4. **Exclusion of Corporate and AI Code:** In strict adherence to project licensing, no code, documentation, or architecture derived from commercial closed-source software or generated via automated artificial intelligence training pipelines may be submitted.
### Submission Protocol
 * **Encrypted Patches:** Send raw git patch files (.patch) or cryptographic bundles directly via encrypted communication channels to the primary author.
 * **No Centralized Forges:** Avoid relying on corporate-hosted git platforms for sensitive protocol adjustments; maintain air-gapped local repositories and verify all hashes independently.

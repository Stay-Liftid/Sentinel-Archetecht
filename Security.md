# Security Policy 
## Reporting a Vulnerability
**The Remote Viewer** enforces a strict private disclosure policy. Public bug bounty platforms, corporate vulnerability brokers, and third-party intermediary services are explicitly rejected.
Any discovered cryptographic flaw, protocol vulnerability, or zero-day exploit within this codebase must be reported exclusively and directly via encrypted channels to the primary author.
### Secure Submission Protocol
 1. **Do Not Open Public Issues:** Public disclosure of unpatched vulnerabilities compromises network integrity and exposes active nodes.
 2. **Encrypt Communication:** Utilize the project's official PGP signing key or verified private channel for transmission.
 3. **Provide Reproduction Steps:** Include clean, localized code or mathematical proof of the flaw without executing active network exploitation.
### Response & Remediation Lifecycle
 * **Direct Acknowledgment:** Receipt verification is issued exclusively via encrypted reply.
 * **Isolated Patching:** The primary author verifies and resolves the vulnerability within the core repository.
 * **Anonymous Attribution:** Researchers acting in good faith receive pseudonymous or anonymous credit within commit logs upon request.
